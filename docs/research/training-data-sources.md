# Training data sources for PokeGrade

_Captured: 2026-06-28._

Goal: source real data to validate and improve PokeGrade's grade prediction, and to power an open verified-returns log. The unit we want is image to official PSA grade. The "AI grade" column is generated by running our own grader over those images; it is for calibration and error analysis, not a training target.

This complements the current Claude-vision grader; it is not a call to replace it with a bespoke model. Even without any model training, image-to-official-grade pairs let us measure how well the existing grader tracks real PSA outcomes and where it drifts.

## The headline

The clean triplet (raw card photo + AI pre-grade + eventual official PSA grade) does not exist as a public dataset. The apps that hold it (SnapGrade reports 50,000+ verified outcomes) treat it as their moat. We assemble it.

## The trap: slabbed images leak the label and do not match our input

Almost every "graded card" image online is the card already sealed in its PSA holder, with the grade printed on the label. The one sizeable public set, `samsilverman/PSA-Baseball-Grades` (11,500 images, 1,150 per grade), is exactly this: the README notes frames include "hands, tables, and PSA grade cases and labels".

Two problems:

1. Label leakage. The printed grade is in the frame, so a model can learn to read it instead of assessing the card.
2. Domain mismatch. Users photograph raw cards before submission, not slabbed cards.

Use raw card face to official grade as the signal. Slab datasets are only a transfer-learning base or card-identification pretraining, never the final grade signal.

## Ranked sources

### 1. PSA Public API - the ground-truth tap (best)

- Endpoint: `GetByCertNumber`, `https://api.psacard.com/publicapi/cert/GetByCertNumber/{cert}`
- Returns PSA's record for a cert number: the card, its official grade, and label details. Free registration token (bearer).
- PSA has attached its own standardised scan images to certs graded since October 2021, so recent certs give a consistent image plus official-grade pair, cleaner than random marketplace photos.
- Ready client: `brad-newman/fetch-psa-api` (Python, pulls cert details and images).
- Caveats: you query by cert number, so you need a list of them (harvest from sold listings, pop aggregators like GemRate, or the Apify PSA scraper). Bulk and dataset use is governed by the PSA API End User Agreement; read it before scraping at scale.
- Docs: https://www.psacard.com/publicapi/documentation

PSA cert scans are still through-the-slab images. They are the best large-scale public proxy for clean card-face images, but the truly clean raw-photo-to-official-grade signal at scale only comes from photographing raw cards and tracking submissions over time, which is what SnapGrade did.

### 2. samsilverman/PSA-Baseball-Grades - transfer base

- 11,500 baseball card images, 1,150 per grade (psa1 to psa10), folder is the label, standardised to 150x200px.
- Overall grade labels only, no sub-grades. Collected via ParseHub from Collectors.com. No explicit licence (treat as non-commercial until clarified).
- Use as a transfer-learning base or methodology template, not as clean Pokémon training data. Slabbed and leaky (see the trap above).
- https://github.com/samsilverman/PSA-Baseball-Grades

### 3. Roboflow Universe - geometry and preprocessing

- `jason-brenan/card-grading`: 41 images annotated for each edge and corner. https://universe.roboflow.com/jason-brenan/card-grading
- `scandex` pokemon-cards segmentation: 109 images for card cropping and deskew. https://universe.roboflow.com/scandex/pokemon-cards-sk39y-4ghz0
- Use to bootstrap centering measurement and card isolation. No grade labels.
- Walkthrough: https://blog.roboflow.com/using-computer-vision-to-make-card-grading-faster-and-cheaper/

### 4. Hugging Face / Kaggle - card images, no grades

- `TheFusion21/PokemonCards` (HF), Kaggle pokemon-cards: card images plus metadata. Good for card identification and segmentation pretraining, not grade labels.
- https://huggingface.co/datasets/TheFusion21/PokemonCards

## Model architecture references

- `crimsonthinker/psa_pokemon_cards`: U-Net to crop the card, then transfer learning (VGG16/ResNet/MobileNet) with separate heads for centering, corners, edges, surface against a `grades.csv`. The dataset is not shared, but the structure is the lesson. https://github.com/crimsonthinker/psa_pokemon_cards
- Automated corner grading of trading cards (peer-reviewed): defect identification with confidence calibration, which maps onto the accuracy goal. https://www.sciencedirect.com/science/article/pii/S0166361524001155

## The build pipeline (assembling the triplet)

1. Harvest cert numbers (sold listings, GemRate, Apify), then pull image plus official grade from the PSA API. Or capture raw photos directly for the cleaner pre-slab signal.
2. Run the grader (current Claude-vision engine, or any model) over those same images to generate the AI-grade column.
3. You now hold image + AI grade + official grade. The misses, the rows SnapGrade hides, are the highest-value calibration and training signal.

## Legal and licensing notes

- PSA API use is governed by the PSA API End User Agreement. Review before bulk collection.
- Collectors.com and marketplace scraping carry their own terms.
- `samsilverman/PSA-Baseball-Grades` has no explicit licence. Do not assume commercial reuse rights.
